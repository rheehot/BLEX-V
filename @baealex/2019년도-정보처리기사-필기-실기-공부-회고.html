다행히 원하던대로 1회차에 필기와 실기를 모두 합격할 수 있었다. 이번에 공부를 해보니 다른 자격증도 뭔가 쉽게 딸 수 있을 것 같은(?) 생각이 들었다. 이 글에서는 필자가 얼마나 공부했고 어떤식으로 공부했고 어떤 점이 후회됐는지 기록해 두고자 한다. 이 자격증에 도전하는 사람들에게 조금이나마 도움이 되었으면 좋겠다.



필기

이 글을 보고 있는 당신의 생각을 존중한다. 만일 단순히 자격증을 소지하는게 목표라면 책은 구매하지 않아도 된다. 기출문제를 쭉 뽑아서 쭉 풀어보면 된다. 그럼 쭉 똑같은 문제가 나올것이고 당신의 시험장에서도 같은 문제가 나올테니까. 하지만 20년에 이 자격증을 준비한다면 책은 반드시 필요하겠다. 참고 : 2020 필기 출제 기준

필자는 '이기 쥬희'라는 유명한 블로거의 기출 문제 모음집을 풀었다. 4일을 계획했으나 건강상의 문제로(공부는 미리미리 적당히 해두자. 무슨일이 생길지 모른다.) 3일간 공부를 진행했고 온전히 집중한 시간은 매일 9시간 정도이다. 시간표는 다음과 같이 선정하였다.

시간표

  • 1일차 : 데이터 베이스 / 전자 계산기
  • 2일차 : 운영 체제 / 전자 계산기
  • 3일차 : 소프트웨어 공학 / 전자 계산기
  • 4일차 : 데이터 통신 / 전 챕터 정리

다른 과목은 바짝 집중하면 금방 문제를 풀어볼 수 있는데다 좀 재밌다. 하지만 전자 계산기는 집중이 잘 안돼서 나눠서 풀었다. 문제를 풀때는 다음과 같은 순서로 풀었다.

공부법

  • 모르는 문제 : 별표치고 개념을 살표본 후 답을 체크하지 않고 넘어감
  • 아는 문제
    • 맞은 문제 : 다시 볼 필요가 없음
    • 틀린 문제 : 틀린 이유와 답을 확인, 확인했고 이해 했으면 세모

한 챕터가 끝났을 때 모르는 문제와 틀린 문제를 쭉 살펴본 후 그래도 잘 모르겠으면 다른 모양으로 표시한다. 시험 보기 전에 그 모양이 표시된 문제만 살펴보고 들어가자.

결과

  • 데이터 베이스 : 90
  • 전자 계산기 : 95
  • 운영 체제 : 85
  • 소프트웨어 공학 : 90
  • 데이터 통신 : 55

실기

실기는 자랑할 만한 점수는 아니지만(69점) 나름 선방했다고 생각한다. 책은 구매하지 않았고 알고리즘과 데이터 베이스 위주로 공부했다. 공부는 필기와 마찬가지로 3일을 진행했으나 2일은 밤샘공부를 진행하였다.

시간표

  • 1일차 : 알고리즘 / 데이터 베이스
  • 2일차 : 알고리즘 / 데이터 베이스
  • 3일차 : 알고리즘 / 데이터 베이스 / 신기술 동향

시간표를 보면 알겠지만 전산영어와 업무 프로세스는 하나도 보지 못했다. 아니 볼 수 없었다. 책이 없기 때문... 상당히 불안 했지만 나름 트렌디한 기술을 많이 안다고 생각하여 알고리즘, 데이터 베이스를 다 맞고 신기술을 맞춰서 합격하고자 하였다. 하지만 신기술은 1문젠가 맞추었다.

공부법

  • 알고리즘 : 알고리즘 초급부터 고급유형을 다 살펴봄
  • 데이터 베이스 : 2007년에 만들어진 자료로 공부함
  • 신기술 동향 : 국가에서 제공하는 자료로 공부함(한 문제도 안나옴)

솔직히 개념 보는거보다 2017년 이후에 출제된 문제를 찾아 보는게 빠르다. (유형을 알아보고 공부하라는 뜻이다.) 시험지는 시험 끝나고 가져가므로 사람들이 복원한 문제를 찾아야 한다. 이 블로그에도 2019 1회 알고리즘 / 데이터 베이스 복원 문제가 하나 있다. 참고 링크에 필자의 공부법이나 공부한 기록이 담겨있다. 참고하시길.

필자는 영어가 매우 잼병인데 전산 영어가 의외로 너무 쉬웠다. 답이 Kernel이나 RAM, Register 같은 컴퓨터 용어가 등장하는 데다 굉장히 문법을 쉽게 구사해 놓았다. 스택 오버플로우를 자주 참고하는 사람이고 그 글을 읽는데 큰 문제가 없다면 큰 걱정하지 않아도 될 듯.

업무 프로세스는 볼 필요가 없다고 하여 정말 쉽게 나올 줄 알았는데 4문제 중에 2문제는 쉽지만 2문제는 책을 구매해야 맞출 수 있을 것 같다. 전문 용어가 나온다. 그래서 결론이 뭐냐하면 알고리즘과 데이터 베이스만 마스터하면 거의 합격한거나 다름없다는 의미다. 좋은 결과 있으시길...
